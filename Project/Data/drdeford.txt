bfs returns generators instead of dicts #TITLE_END#The bfs and dfs traversals return different object types for their predecessors and successors. The documentation [here](https://networkx.github.io/documentation/networkx-1.10/reference/generated/networkx.algorithms.traversal.breadth_first_search.bfs_predecessors.html#networkx.algorithms.traversal.breadth_first_search.bfs_predecessors ) suggests that they should all return dicts but bfs returns a generator of the edges:  ``` >>> G=nx.Graph() >>> G.add_path([0,1,2]) >>> print(nx.dfs_predecessors(G,0)) {1: 0, 2: 1} >>> print(nx.bfs_predecessors(G,0)) <generator object bfs_predecessors at 0x000002141B81E518> >>> print(nx.dfs_successors(G,0)) {0: [1], 1: [2]} >>> print(nx.bfs_successors(G,0)) <generator object bfs_successors at 0x000002141B81E518>  ```
