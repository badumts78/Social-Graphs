MultiDigraph sets edge id=0 globally when creating a graphml file#TITLE_END#I'm trying to export the G graph as a graphml file so that I can later on open that plot using yEd.  The network that I'm creating is the following.  ``` G = nx.MultiDiGraph()  G.add_node('a', label='a') G.add_node('b', label='b')  G.add_edge('a','b', label='a-to-b') G.add_edge('b','a', label='b-to-a')  nx.write_graphml(G,'test.graphml') ```  ### Current Behavior  When I inspect the produced `test.graphml` I observe that each edge section has a value `id=0`, such as:  ``` <edge source="a" target="b" id="0">   <data key="d1">a-to-b</data> </edge> <edge source="b" target="a" id="0">   <data key="d1">b-to-a</data> </edge> ``` The above result makes yEd consider the last edge only when plotting the network.  ![image](https://github.com/user-attachments/assets/dcab7f1b-3f9f-4690-a488-0aeb8a35b548)  ### Expected Behavior  If I manually change the Id of the edges to be unique in the network, then I can plot all of the edges, such as:  ``` <edge source="a" target="b" id="0">   <data key="d1">a-to-b</data> </edge> <edge source="b" target="a" id="1">   <data key="d1">b-to-a</data> </edge> ```  ![image](https://github.com/user-attachments/assets/61a141b5-4b71-4aca-bafd-e1ab303c27f8)  ### Steps to Reproduce  In order to reproduce, follow the code above.  ### Environment  I'm using this with  Ubuntu 22.04 Python version: 3.10.12 NetworkX version: 3.4.2 yEd: 3.24
