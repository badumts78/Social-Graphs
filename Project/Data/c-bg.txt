Setting per-node attributes with `add_nodes_from` does not set attributes#TITLE_END#Thanks a lot for maintaining this package, it's great! :)  I've witnessed a potential bug and would like to contribute a bug report.   When I set attributes with `add_nodes_from`, and then look at the nodes with `g.nodes(data=True)`, I do not see the attributes I specified. If I set them via `nx.set_node_attributes`, I can later see them.  This is contrary to how one can add attributes to edges.   ### Current Behavior <!--- Tell us what happens instead of the expected behavior --> ```python import networkx as nx h = nx.MultiDiGraph() nodes_with_attrs = {'A': {'color': 'yellow'}, 'B': {'color': 'blue'}, 'C': {'color': 'red'}} h.add_nodes_from(nodes_with_attrs) h.nodes(data=True) ``` results in output (1) ``` NodeDataView({'A': {}, 'B': {}, 'C': {}}) ``` Additionally running  ```python nx.set_node_attributes(h, nodes_with_attrs) h.nodes(data=True) ``` results in output (2) ``` NodeDataView({'A': {'color': 'yellow'}, 'B': {'color': 'blue'}, 'C': {'color': 'red'}}) ```  ### Expected Behavior <!--- Tell us what should happen --> Output (1) should already be output (2), I think.   This would also be consistent with the behavior for edges:  ```python import networkx as nx h = nx.MultiDiGraph() h.add_nodes_from('A B C'.split()) edges_with_attrs = [('A', 'B', {'capacity': 1}), ('B', 'C', {'capacity': 2}), ('C', 'A', {'capacity': 3})] h.add_edges_from(edges_with_attrs) h.edges(data=True) ``` results in output ``` OutMultiEdgeDataView([('A', 'B', {'capacity': 1}), ('B', 'C', {'capacity': 2}), ('C', 'A', {'capacity': 3})]) ```  ### Steps to Reproduce <!--- Provide a minimal example that reproduces the bug --> (cf. the code example above)  ### Environment <!--- Please provide details about your local environment --> Python version: 3.8.2 NetworkX version: 2.5.1   Thanks much for your feedback & help in advance!
