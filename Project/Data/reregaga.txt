"Error: node, position, expected two doubles" when using AGraph renderer#TITLE_END#### Current Behavior  Get error about `Error: node A, position (0, 0), expected two doubles`: ``` lib\site-packages\pygraphviz\agraph.py:1402: RuntimeWarning: Error: node A, position (0, 0), expected two doubles Error: node B, position (1, 2), expected two doubles ``` if run code: ```python import networkx as nx  # Create Graph Gr = nx.Graph() Gr.add_node('A', pos=(0, 0)) Gr.add_node('B', pos=(1, 2))  # Create AGraph Ag = nx.nx_agraph.to_agraph(Gr)  # Output Ag.write('out.dot') Ag.layout('neato') Ag.draw('out.png') ```  ### Expected Behavior  Without errors.  ### Steps to Reproduce #### Without converting `pos` I get this error and image is not correct Code: ```python import networkx as nx  # Create Graph Gr = nx.Graph() Gr.add_node('A', pos=(0, 0)) Gr.add_node('B', pos=(1, 2))  # Create AGraph Ag = nx.nx_agraph.to_agraph(Gr)  # Output Ag.write('out.dot') Ag.layout('neato') Ag.draw('out.png') ``` Result `out.dot` file: ``` strict graph "" { 	A	[pos="(0, 0)"]; 	B	[pos="(1, 2)"]; } ``` Rendered image: ![out](https://user-images.githubusercontent.com/76629445/195459562-bcac373b-1ba1-4bb4-b7dc-b489161cdde4.png)  #### With manually converting `pos` I don't get error and image is correct With code: ```python import networkx as nx  # Create Graph Gr = nx.Graph() Gr.add_node('A', pos=(0, 0)) Gr.add_node('B', pos=(1, 2))  # Convert `"(x,y)"` to `(x,y!)` for node in Gr:     Gr.nodes[node]['pos'] = "{},{}!".format(         Gr.nodes[node]['pos'][0], Gr.nodes[node]['pos'][1])  # Create AGraph Ag = nx.nx_agraph.to_agraph(Gr)  # Output Ag.write('out.dot') Ag.layout('neato') Ag.draw('out.png') ``` Result `out.dot` file: ``` strict graph "" { 	A	[pos="0,0!"]; 	B	[pos="1,2!"]; } ``` Result image file: ![out](https://user-images.githubusercontent.com/76629445/195459409-037b104a-a829-4a9f-8c49-a024be5483cd.png)   ### Environment  Python version: `3.9.1` NetworkX version: `2.8.7`  ### Additional info Maybe I'm missing the right way to use `AGraph`, but now I noticed that `AGraph` writes a node position into the file `out.dot` in a format that does not correspond to the [DOT](https://graphviz.org/docs/attr-types/point/) format (`pos="(0,0)"` vs `pos="0,0"`), so I have to manually convert the position format, otherwise I will get an error and incorret `out.dot` and `out.png` files.
