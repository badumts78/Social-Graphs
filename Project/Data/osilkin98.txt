Incorrect clustering coefficient calculation#TITLE_END#The clustering coefficient is calculated incorrectly for multiple of the terms. This is due to the fact that computing the number of edges isn't correct in the closed neighborhood for a node u âˆˆ G.  Consider the following graph G: ![image](https://user-images.githubusercontent.com/16809287/66355646-fdaca780-e935-11e9-85fb-ce1149b25cb8.png)   In the above graph, the clustering coefficient is computed as follows:  ```py >>> edges = [('a', 'b'), ('a', 'c'), ('a', 'e'), ('b', 'c'), ('c', 'e'), ('c', 'd'), ('d', 'e')] >>> G = nx.Graph(edges) >>> nx.clustering(G) {'a': 0.6666666666666666, 'b': 1.0, 'c': 0.5, 'e': 0.6666666666666666, 'd': 1.0} ``` However, 'a' and 'd' should both be `5/6`, and `c` be `7/10`  The following computation calculates the correct clustering coefficient: ```py def clustering_coefficient(G: nx.Graph, *nodes):     nodes = nodes or G.nodes     clusters = {}     for v in nodes:         # Here we calculate the number of edges contained          # in the subgraph induced by v and its adjacent neighbors         closed_hood = list(G.neighbors(v)) + [v]            induced = nx.induced_subgraph(G, closed_hood)            # The most edges that the induced subgraph can have is: deg(v) * (deg(v) - 1) / 2           max_edges = len(induced) * (len(induced) - 1) / 2                  # Thus we compute cc(v) = |E(N[v])| / |E(K_deg(v)+1)|         clusters[v] = len(induced.edges) / max_edges      return clusters ```  And we see that:  ```py >>> clustering_coefficient(G) {'a': 0.8333333333333334, 'b': 1.0, 'c': 0.7, 'e': 0.8333333333333334, 'd': 1.0}  >>> cc_G = sum(cc for cc in clustering_coefficient(G).values())/len(G) 0.8733333333333334  >>> true_cc = (5/6 + 5/6 + 1 + 1 + 7/10) / 5 >>> math.isclose(true_cc, ccG) True ```
