Fix weakly_connected_components() performance on graph view#TITLE_END#Description ======== weakly_connected_components() calls len(G) inside a loop, resulting in poor performance on graph views. suggested pull request moves len(G) to outside the loop.  Reproduction: =========  ``` import datetime import os.path import sys  sys.path.insert(0, os.path.dirname(__file__)) from networkx import DiGraph, weakly_connected_components, subgraph_view  g = DiGraph() for i in range(10_000):     g.add_node(i)  view = subgraph_view(g, filter_node=lambda n: True, filter_edge=lambda u, v: True)  for i in range(10):     start = datetime.datetime.now()     for c in weakly_connected_components(view):         pass     end = datetime.datetime.now()     tmp = None     print(f'{(end - start).total_seconds():.9f}') ```  Evaluation results: =========== * before fix: ~15.4sec per iteration * after fix: 0.064sec per iteration
