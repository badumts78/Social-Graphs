maximal_matching and max_weight_matching have different matchings on unweighted graph#TITLE_END#On this simple, unweighted graph, maximal_matching and max_weight_matching return different sizes of matchings. A maximal matching would be (a,f), (c,d) and (b,e), however, the maximal_matching function only finds two matchings instead of three.  ``` python >>> import networkx as nx >>> g = nx.Graph() >>> g.add_edges_from([('a','e'),('a','d'),('a','f'),('b','c'),('b','e'),('c','d'),('c','e'),('e','f')]) >>> nx.max_weight_matching(g) {'a': 'f', 'c': 'd', 'b': 'e', 'e': 'b', 'd': 'c', 'f': 'a'} >>> nx.maximal_matching(g) set([('c', 'b'), ('a', 'e')]) ```   ![figure_1](https://cloud.githubusercontent.com/assets/7592559/24074645/3227f790-0be3-11e7-95bc-90a11adcf5e8.png)
