node_link_data from readwrite.json_graph #TITLE_END#I think I stumbled upon a bug in `node_link_data` function from `networkx.readwrite.json_graph `. Below the minimal example of what I mean . ``` import networkx.readwrite.json_graph as jg import networkx as nx G = nx.Graph() G.add_nodes_from([3,4,5,6,7]) G.add_edge(3,4) G.add_edge(4,5) d = jg.node_link_data(G) d["links"] ``` the return of it is `[{'source': 0, 'target': 1}, {'source': 1, 'target': 2}]`. The nodes are stored with their original id.   (I use MacOs (10.11.5), python 2.7 and updated all packages to current pip version)
